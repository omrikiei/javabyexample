Java by Example: Exceptions

Java uses exceptions for error handling. Exceptions are objects that represent errors and can be thrown and caught.

 
// Custom exception by extending Exception.
class ValidationException extends Exception {
    private final int value;

    ValidationException(String msg, int value) {
        super(msg);
        this.value = value;
    }

    int getValue() {
        return value;
    }
}

Methods declare checked exceptions with throws.

 
public class Exceptions {

    static int divide(int a, int b) throws ArithmeticException {
        return a / b;
    }

    static void validate(int age) throws ValidationException {
        if (age < 0) {
            throw new ValidationException("negative age", age);
        }
    }

Use try/catch/finally to handle exceptions.

 
    public static void main(String[] args) {
        // Catching a runtime exception
        try {
            divide(10, 0);
        } catch (ArithmeticException e) {
            System.out.println("error: " + e.getMessage());
        }

        // Catching a custom exception
        try {
            validate(-5);
        } catch (ValidationException e) {
            System.out.println("invalid: " + e.getMessage()
                + " (value=" + e.getValue() + ")");
        }

        // try-with-resources for auto-closeable resources
        // (similar to Go's defer)
        System.out.println("done");
    }
}
 
$ java Exceptions.java
error: / by zero
invalid: negative age (value=-5)
done
