Java by Example: Sealed Classes

Sealed classes (Java 17+) restrict which classes can extend them. This enables exhaustive pattern matching and better domain modeling.

 
// A sealed interface can only be implemented
// by permitted classes.
sealed interface Shape permits Circle, Rectangle {
    double area();
}

Each permitted subclass must be final, sealed, or non-sealed.

 
final class Circle implements Shape {
    double radius;
    Circle(double r) { this.radius = r; }
    public double area() { return Math.PI * radius * radius; }
}

final class Rectangle implements Shape {
    double w, h;
    Rectangle(double w, double h) { this.w = w; this.h = h; }
    public double area() { return w * h; }
}

Pattern matching with sealed types (Java 21+).

 
public class SealedClasses {
    static String describe(Shape s) {
        return switch (s) {
            case Circle c -> "Circle with radius " + c.radius;
            case Rectangle r -> "Rectangle " + r.w + "x" + r.h;
        };
    }

    public static void main(String[] args) {
        Shape c = new Circle(5);
        Shape r = new Rectangle(3, 4);
        System.out.println(describe(c));
        System.out.println(describe(r));
        System.out.println("area: " + c.area());
    }
}
 
$ javac SealedClasses.java
$ java SealedClasses
Circle with radius 5.0
Rectangle 3.0x4.0
area: 78.53981633974483
