Java by Example: Atomic Counters

For managing simple shared state, Java provides java.util.concurrent.atomic package with lock-free thread-safe operations.

 
import java.util.concurrent.atomic.AtomicLong;

public class AtomicCounters {
    public static void main(String[] args) throws Exception {
        var ops = new AtomicLong(0);

We’ll start 50 threads that each increment the counter 1000 times.

 
        var threads = new Thread[50];
        for (int i = 0; i < threads.length; i++) {
            threads[i] = new Thread(() -> {
                for (int j = 0; j < 1000; j++) {
                    ops.incrementAndGet();
                }
            });
            threads[i].start();
        }

Wait for all threads and read the final value.

 
        for (var t : threads) t.join();

        System.out.println("ops: " + ops.get());
    }
}

We expect exactly 50,000 operations. Atomics are often preferred over locks for simple counters.

 
$ java AtomicCounters.java
ops: 50000
